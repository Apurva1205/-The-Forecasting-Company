How I would approach this role

1. Ground everything in data & baselines
  • Start by instrumenting robust baselines (seasonal naive, simple regressors, classical models) across key customer datasets to get a clean reference.
  • Build exploration notebooks & tools for lag analysis, cross‑correlations between data sources (e.g., weather vs sales), and sparsity patterns.
2. Iterate on foundation model architectures
  • Implement and combine ideas from state‑of‑the‑art time‑series papers (sequence models, transformers for time series, hierarchical/context‑aware approaches).
  • Treat each change as an experiment: clear config, logged metrics, stored checkpoints, and comparable runs.
3. Make it production‑ready
  • Work closely with the team to export models (e.g., ONNX or torch.compile) and debug any low‑level issues that show up when moving from research to API.
  • Add diagnostics & accuracy reports that users can see alongside forecasts, not hidden in logs.
4. Learn from customer feedback
  • Use feedback (“this lag feels wrong”, “this regime shift wasn’t captured”) to update training data, loss functions, and model capabilities.
  • Help create playbooks for new industries so onboarding the next customer gets faster.
